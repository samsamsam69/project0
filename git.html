So, What's Git?

Git a program that runs through the terminal (cmd in Windows). Git is a version control software - it can track changes in the files of a repository. This can be helpful to programmers, as they can revert back to an earlier version of the software if they need to.

Here are some git commands that are frequently used:

Commands Function
git clone your_url Clones the repository from the URL into the specified local directory
git add file_name Tells Git to track files that are added
git commit -m "your_message" Commits (saves) the files that were previously added. the "-m "your_message"" allows us to 'label' the commits, so when referred back, we can see what changes are made in which commits
git init Initialises Git when making new folder locally
git branch Git will show all branches in the repository
git branch branch_name Makes new branch with a the specified branch_name
git checkout branch_name "Selects" the specified branch_name. This changes made to the files will then take place in this branch, not others
git status Checks for the status of the repository: if any files need to be committed, or the repository is up-to-date to the git
git log Shows the history of commits of the repository
